# ネズミ被害 相談ガイド LP（屋根裏・天井裏）

A8.net 害獣駆除案件向けの広告LPです。静的HTML/CSS/JavaScriptのみで構成されており、
外部ライブラリ・CDN・写真素材を一切使わず、SVGイラスト・SVGアイコンのみで
構成されています。

## 1. ファイル構成

```
nezumi-lp/
├── index.html        ... LP本体（1ページ完結）
├── styles.css         ... デザイン（スマホファースト）
├── script.js          ... A8リンク設定・CTAクリック計測
├── README.md          ... このファイル
└── assets/            ... SVGイラスト・アイコン
    ├── hero-house.svg
    ├── symptom-sound.svg
    ├── symptom-droppings.svg
    ├── risk-wiring.svg
    ├── prevention-entry.svg
    ├── merit-inspection.svg
    ├── flow-steps.svg
    ├── cta-consult.svg
    ├── self-check.svg
    ├── damage-signs.svg
    ├── cost-factors.svg
    └── check-points.svg
```

`assets/` 内のSVGが万一見つからない場合でも、`<img>` の `onerror` で非表示になり、
CSSのグラデーション＋絵文字アイコンのプレースホルダーが表示されるため、
表示崩れは起きません。

## 3. A8_LINK の差し替え方法

`script.js` の冒頭にある以下の変数を、取得したA8.netの広告リンクURLに書き換えてください。

```js
const A8_LINK = "#";
```

例：

```js
const A8_LINK = "https://px.a8.net/svt/ejp?a8mat=XXXXXX...";
```

この変数を1箇所書き換えるだけで、ページ内すべてのCTAボタン（`data-cta`属性付きの
リンク）が同じリンク先に自動設定されます。`index.html` 側のCTAは
すべて `href="#"` のままでよく、修正不要です。

## 4. GA4タグの入れ方

`index.html` の `<head>` 内、コメント `<!-- GA4 / GTM 設置欄 -->` の箇所に
GA4の発行タグをそのまま貼り付けてください。

```html
<script async src="https://www.googletagmanager.com/gtag/js?id=G-XXXXXXXXXX"></script>
<script>
  window.dataLayer = window.dataLayer || [];
  function gtag(){dataLayer.push(arguments);}
  gtag('js', new Date());
  gtag('config', 'G-XXXXXXXXXX');
</script>
```

`script.js` はページ内に `gtag` 関数が存在する場合、CTAクリック時に自動で
`affiliate_cta_click` イベントを送信します（追加のコード変更は不要）。

## 5. GTMタグの入れ方

GTMコンテナを使う場合は、GTM発行のスクリプトタグ（`<head>`用・`<body>`開始直後用の
両方）を、`index.html` 内の該当箇所に貼り付けてください。

GTM経由でGA4イベントを発火させる場合も、`script.js` から送られる
`affiliate_cta_click` イベント（`dataLayer.push`相当）をトリガーとして
GTM側でタグ設定できます。

## 6. Google広告コンバージョンタグの入れ方

`index.html` の `<head>` 内、コメント `<!-- Google広告 コンバージョンタグ設置欄 -->` の
箇所にコンバージョンイベントのコードを追記してください。

`script.js` のCTAクリック処理内にも、コンバージョンタグ送信用のコメントアウト箇所を
用意しています。コメントを外し、`send_to` の値を実際のIDに差し替えて有効化してください。

```js
// script.js 内（コメントを外して使用）
if (typeof gtag === "function") {
  gtag("event", "conversion", {
    send_to: "AW-XXXXXXXXX/XXXXXXXXXXXXXXXXXXXX"
  });
}
```

## 7. CTAクリック計測の仕組み

`script.js` は、`data-cta` 属性を持つすべてのCTAボタンに対して以下を行います。

1. `href` を `A8_LINK` の値に設定
2. クリック時に `console.log("affiliate_cta_click", { position: ... })` を出力（開発確認用）
3. `gtag` が読み込まれていれば、以下の内容でイベントを送信

```js
gtag("event", "affiliate_cta_click", {
  event_category: "affiliate",
  event_label: "a8_link_click",
  cta_position: position
});
```

`cta_position`（`data-cta` の値）は以下のいずれかです。

```
fv, checklist, selfcheck, risk, selfcare, costfactors, merit, checkpoints, flow, faq, fixed, final
```

`selfcheck`（3分セルフチェック）、`costfactors`（費用が変わる理由）、
`checkpoints`（相談前に確認しておきたいポイント）は3回目改修で追加した
セクションのCTAです。`script.js` は `[data-cta]` を持つ要素を汎用的に
処理するため、新しい `data-cta` 値を追加してもコード変更は不要です。

各CTAがどのセクションでクリックされたかをGA4側でレポートできます。

## 12. SVGの差し替え方法

各SVGは `assets/` 内の同名ファイルを上書きするだけで差し替えられます。
`index.html` 側の `<img src="assets/xxx.svg">` の修正は不要です。

- SVGの `viewBox` は概ね `0 0 400 170`（ファーストビューのみ `0 0 400 200`）で
  統一しています。差し替える際も同程度の比率（横:縦 ≒ 2.3〜2.4:1）にすると
  `.media-frame` の枠内にきれいに収まります。
- `.media-frame img` は `object-fit: contain` を使用しているため、
  比率が多少異なってもレイアウト崩れは起きません（余白ができるのみ）。
- ファイル名を変更する場合は、`index.html` 内の該当 `<img src="...">` も
  あわせて変更してください。
- 画像が見つからない場合は `onerror` により非表示になり、
  `.media-frame` のグラデーション＋絵文字プレースホルダーが表示されます。

## 13. 画像制作方針

- 写真素材・外部画像は使用しない（SVGイラスト・アイコンのみ）
- リアルなネズミの姿、糞や死骸、捕獲シーンなど不快感の強い描写は避ける
- ネズミを描く場合も、ごく小さなシルエット程度にとどめる（`hero-house.svg`）
- 清潔感・安心感・相談しやすさを重視した配色（緑・白・薄いベージュ・オレンジ）で統一
- 画像内のテキストは最小限（ステップ番号のみ）にし、説明文はHTML側で管理する
- 40〜70代でも直感的に理解できる、シンプルな図形・アイコンを使用する

## 13-2. 3回目改修（CVR改善）で追加したセクション

CVR改善（自分ごと化・費用不安の解消・相談への不安解消）を目的に、
以下の4セクションを追加し、ページ構成は以下の順番になっています。

1. ファーストビュー（`#fv`）
2. 症状チェック（`#checklist`）
3. **3分セルフチェック（`#selfcheck`）** ※追加
4. **被害サインのカード（`#damage-signs`）** ※追加
5. 放置するリスク（`#risk`）
6. 自力対策では難しい理由（`#selfcare`）
7. **費用が変わる理由（`#cost`）** ※追加
8. 専門業者に相談するメリット（`#merit`）
9. **相談前に確認しておきたいポイント（`#checkpoints`）** ※追加
10. 相談から訪問日程調整までの流れ（`#flow`）
11. 対応エリアに関する注意（`#area`）
12. よくある質問（`#faq`）※3項目追加
13. 最終CTA（`#final`）
14. 広告表記・免責（フッター）

### 各セクションの目的

| セクション | 目的 |
|---|---|
| `#selfcheck`（3分セルフチェック） | 7項目のチェックリストでユーザーに「自分ごと化」を促し、2つ以上該当する場合は早めの相談を提案する |
| `#damage-signs`（被害サインのカード） | 物音・糞・臭い・かじり跡・シミ・侵入口の6サインをカード形式で提示し、症状への気づきを後押しする |
| `#cost`（費用はなぜ変わるの？） | 「ネズミ 駆除 費用」「相場」等の検索意図に応える形で、費用が変動する一般的な要因を説明し、費用面の不安を和らげる |
| `#checkpoints`（相談前に確認しておきたいポイント） | 業者に相談する際の一般的な確認ポイントを提示し、相談への心理的ハードルを下げる |
| FAQ追加3項目 | 「費用が事前にわかるか」「市販グッズで様子見してよいか」「天井のシミ・臭いも相談対象か」という疑問に答え、相談への後押しとする |

### 表現方針について（重要）

- 上記の追加セクションは、いずれも**ROY公式の固有サービス内容や診断結果として
  断定するものではなく**、一般的なネズミ被害の悩み・相談前の確認事項として
  記載しています。
- **費用・対応スピード・保証・実績・無料・全国対応・再発防止の確約・完全駆除・
  最安・追加料金なし**については、本セクションも含めLP全体で断定的な表現を
  使用していません。費用や対応内容に関する記述は、すべて
  「状況により異なります」「可能性があります」「早めの相談がおすすめです」
  「正確な費用や対応内容は遷移先ページ・現地確認時にご確認ください」といった
  表現で統一しています。
- セクション構成・訴求の検討にあたり競合LPの構成（セルフチェック、費用解説、
  業者選びのポイント等）を参考にしていますが、**害虫駆除への訴求、自前の
  電話番号・問い合わせフォームの設置、公式サイトと誤認させる表現は一切
  採用していません**。
- 3分セルフチェックはチェックボックス風の見た目（クリックで☑表示が切り替わる
  だけの演出）であり、**送信機能・フォーム・入力欄は一切持ちません**
  （`script.js` の `is-checked` クラス切り替え処理のみ）。
- FV・最終CTA直下には「※北海道・東北地方は対象外です。対応可否・費用・調査内容は
  遷移先ページでご確認ください。」という注意書き（`.cta-disclaimer`、11px）を
  追加しています。他のCTA（途中セクション）には付けず、クリック率への影響を
  抑えています。

## 15. 広告審査・A8条件で注意する表現

### 表現面で注意するポイント

- 「公式」「公式サイトへ」「公式申込」「正規」など、公式と誤認させる表現は使用しない
- 「必ず駆除」「完全駆除」「絶対」「業界No.1」「最安」など断定的な表現は使用しない
- 不安を過度に煽る表現（恐怖訴求の強い文言・リアルすぎる被害描写）は避ける
- 料金は「費用は被害状況や作業内容により異なります」と表現し、断定しない
- 「無料」は多用しない。使用する場合は条件確認が必要である旨を添える
- 社名・サービス名・固有の商標（例：ROY、ロイ、ROY株式会社、害獣駆除のROY等）は
  本文中で使用・強調しない
- 害虫（シロアリ・ゴキブリ・ハチ・ダニ・ノミ等）への直接的な訴求はLP本文から除外する

### A8条件違反になりやすいポイント（要確認）

- 北海道・東北地方からの問い合わせを誘導する表現
- 害虫駆除（シロアリ・ゴキブリ・ハチ・ダニ・ノミ等）への誘導
- ホームページ経由の申込みを促すような表現・自前フォームの設置
- 自前の電話番号を掲載すること
- 重複・いたずら・連絡不通につながるような誘導（過度な煽り文句など）
